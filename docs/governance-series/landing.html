Framework Essays — Concept Work

Governing
LLM-Assisted Work

Most advice about using LLMs in knowledge work focuses on better prompts. These essays focus on what better prompts cannot fix.

When an LLM is doing iterative intellectual labor on a long-lived project — drafting, revising, auditing, proposing alternatives — a specific failure regime emerges that is invisible from inside a single conversation. Documents get cleaner. Scope gets tighter. Everything reads as progress. But the same substantive defects persist in relabeled form, the project quietly becomes a different project, and there is no structural mechanism to detect any of it.

This is a governance problem, not a prompting problem. These three core essays describe the framework, the document pattern, and the workflow architecture developed to address it — derived from direct application on a live research program, not from theoretical preferences. A companion architecture note maps the same system as components and implementation status.

Designing Multi-Role AI Collaboration Workflows

The default AI collaboration posture — one context, one conversation, handling everything — produces closed-loop validation at the workflow level. This essay describes a four-role architecture developed on a live project: a research layer, a buill layer, an ideation layer, and a separate authorization gate that is the only context permitted to commit outputs to the project record. Each role is defined by what it cannot do as much as what it can.

What these pieces are doing

What the series argues

Long-lived AI-assisted work needs governance infrastructure, not just better prompts or more iteration. The series argues for three complementary layers: a control framework that defines the governance problem, a document pattern that stabilizes project identity across revision, and a multi-role workflow architecture that breaks closed-loop validation at the process level.

Where this came into focus

The framework was not built as an abstract exercise. It emerged while attempting to keep a live research program stable across multiple audit cycles, meaningful revisions, and repeated AI-assisted attempts at problem definition. The source material predates the essays.

What makis this distinctive

Most writing about AI-assisted work focuses on prompting, output quality, or evaluation. These pieces focus instead on how to preserve project identity, bound revision, separate authority, and judge convergence over time — the things that must be correct before an AI-assisted program can be said to be well-governed.

Where this fits

The adjacent literatures are LLM auditing, research proposal validation, qualitative convergence theory, and weak supervision frameworks. This work sits between them rather than inside any one of them — the combination of frozen identity anchor, persistent defect register, and cross-cycle convergence judgment does not appear to exist as a packaged framework elsewhere.
