A Control Framework for LLM-Assisted Research

Governing Research Definition Under Weak Supervision

The central problem is not that LLM-assisted research produces errors. It is that certain errors harden into structure while appearing to improve the work.

Most writing about AI-assisted research focuses on output quality: better prompts, better models, better retrieval. Those matter, but they do not address the deeper governance problem. A solo researcher using an LLM across many cycles of revision can end up with a project that is cleaner, narrower, and more internally coherent than before — yet no longer the same project in a substantive sense.

The failure mode is structural. A false premise becomes a working assumption; a working assumption becomes a design constraint; a design constraint becomes a governing rule. By the time the researcher notices drift, the artifact set may already encode it as if it had always been part of the project.

A governance system for LLM-assisted research must judge movement across cycles, not just apparent quality inside any single draft.

The control-theory framing

The framework treats the problem as a control problem. You need a target state, an observed current state, a way to detect error, a bounded way to apply correction, and a way to decide whether the system is converging or merely circling. That is more demanding than ordinary revision, but it is also the minimum needed for long-lived research work under weak supervision.

Reference signal

A frozen statement of what the research program is and what must not be lost.

Governed object

The current proposal / pre-registration bundle being audited and revised.

Sensors

An adversarial audit instrument that detects defect structure rather than polishing prose.

Controller

A bounded revision process constrained by registered defects, not freeform improvement.

The requirements

A workable governance system needs more than review prompts. It needs an explicit identity anchor, an authority hierarchy across artifacts, adversarial audit discipline, persistent defect memory, bounded patch generation, re-audit, and a convergence layer capable of terminal-state judgments. The important point is that these requirements are derived from the failure modes of the process, not from stylistic preference.

Identity preservation

The project needs a frozen description of what must remain invariant across cycles.

Authority control

The system must distinguish current governing artifacts from historical calibration artifacts.

Defect persistence

Problems must remain identifiable across cycles instead of being forgotten or rephrased away.

Bounded revision

Changes must be tied to defects rather than justified by generic improvement language.

Decision states

The convergence layer should be able to output only a small number of valid terminal or interim states. These are not moods or editorial impressions; they are governance decisions.

Continue

Real defects remain and the revision path is still viable.

Commit

Blocking defects are exhausted and the bundle is ready to bind.

Narrow

A reduced scope still preserves the project’s identity anchor.

Pause

Design work is too unresolved for further revision to mean much.

Kill

The current form cannot be made commitment-ready without unacceptable drift.

Why this matters

This framework is aimed at people doing serious knowledge work with LLMs: technical writers moving into AI-adjacent roles, AI practitioners designing research workflows, and independent researchers trying to preserve methodological discipline while still using probabilistic tools. The point is not to eliminate iteration. It is to stop treating iteration as self-validating.

This piece is a publication-ready framework article extracted from a larger governance program and written as the systems paper for the series.
